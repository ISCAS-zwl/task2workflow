请将用户输入的task拆分为一组"可执行子任务",并以json格式保存,这些子任务将通过 LangGraph 工作流运行，因此必须满足以下要求：
##重要要求##
你必须只输出纯 JSON。不要输出任何自然语言、说明、评论或额外文本。
最终输出必须只包含一个 JSON 对象，不允许出现其他内容。

##Schema 硬性要求(必须严格遵守)##

1. 顶层结构必须是：
{{
  "nodes": [...],
  "edges": [...]
}}

2. 节点(node)要求:
- id 必须是 "ST{{数字}}"，例如 ST1、ST2 ...
- 编号必须从 1 开始，且连续，不允许跳号。
- executor 如果为 "tool"，则 tool_name 必须是工具列表中的名称（完全一致）。
- executor 如果为 "llm"，则 tool_name 必须为 null。
- 当字段无值时必须使用 null（而不是字符串 "null"）。

3. edges 列表：
每个元素必须是：
{{ "source": "STx", "target": "STy" }}

4. JSON 必须为严格标准 JSON:
- 不允许出现注释
- 不允许出现尾随逗号
- 不允许多余字段、空字段或格式错误

##子任务可执行性要求##
1. 每个子任务必须是工具或 LLM 能够独立完成的操作

2. 每个子任务必须包含：
    - 唯一标识 ID (格式为 "ST1", "ST2", ...)
    - 任务名称
    - 任务描述(LLM 或工具如何执行)
    - executor("llm" 或 "tool")
    - tool_name(如果 executor="tool",必须指定工具名称,否则为null)
    - 前置任务(如果没有则为null)
    - 后置任务(如果没有则为null)    
    - 输出内容(子任务预期输出结果，用于后续核对节点执行结果)
    - 输入内容(executor="tool" 时需要指定工具输入参数;为LLM时,输入内容为提示词)

3. 每个子任务之间的依赖必须形成 DAG(无环)，用于构建工作流图。

##引用上游输出的规则##
1. 当需要引用上游节点的输出时，只能使用格式：{{STx.output}}
2. **禁止**使用复杂路径，如 {{STx.output[0].field}} 或 {{STx.output.field}}
3. 系统会自动通过参数整形节点处理字段匹配和类型转换
4. 例如：
   - 正确：{{"param": "{{ST1.output}}"}}
   - 错误：{{"param": "{{ST1.output[0].stationCode}}"}}
